## Discussion Questions (to engage with comments)

### "This looks amazing! What's your favorite feature?"
"My favorite is the multi-agent system. Being able to spawn a browser agent to research something while a coding agent implements it ‚Äì all in parallel ‚Äì is a game changer for complex tasks."

### "How does this compare to Cursor / Windsurf?"
"Great question! Navi is focused on being a GUI for Claude Code specifically, with a few key differences:
1. Multi-agent coordination ‚Äì parallel AI agents with visual hierarchy
2. Local-first ‚Äì your data never leaves your machine
3. Highly extensible ‚Äì Skills system lets you add custom capabilities
4. Open source ‚Äì free and MIT-licensed"

### "Will there be a Windows version?"
"Yes! Windows support is planned. Currently Navi runs on macOS and Linux. We're working on Windows compatibility ‚Äì follow the repo for updates!"

### "Can I use this with other AI providers?"
"Right now Navi is optimized for Claude via the claude-agent-sdk. We're exploring support for other providers in the future. If you're interested in contributing multi-provider support, we'd love the help!"

### "What's the tech stack?"
"Svelte 5 for the frontend (new runes syntax is great!), Tauri v2 for the desktop wrapper, and Bun for the backend server. The database is SQLite (sql.js) and we use the claude-agent-sdk for AI."
